Chapter 1

Introduction

简介

IdealOne OMS is a multi-tenant Order Management System designed for e-commerce operators. It syncs orders from multiple sales platforms, tracks fulfillment status, manages inventory, and provides clients with a branded order portal.

Who uses IdealOne OMS?

  • Administrators — Warehouse and operations staff who manage connections, sync orders, fulfill shipments, and adjust inventory.
  • Clients — Merchants whose orders are being fulfilled. They log into the Client Portal to track their order status and download reports.

System Architecture

Each client account (tenant) has its own isolated database. The administrator OMS is accessed at / and the client portal at /portal.

IdealOne OMS 是一个多租户订单管理系统，专为电商运营商设计。它可以从多个销售平台同步订单、跟踪发货状态、管理库存，并为客户提供品牌化订单门户。

用户角色

  • 管理员 — 仓库和运营人员，负责管理连接、同步订单、履行发货及调整库存。
  • 客户 — 使用我们服务的商家，通过客户门户登录，查看订单状态和下载报表。

系统架构

每个客户账户（租户）都有独立的数据库。管理员 OMS 通过 / 访问，客户门户通过 /portal 访问。

Chapter 3

Dashboard

主页

The Dashboard is your main workspace. It shows all orders, key statistics, and platform connection status at a glance.

Stats Cards

At the top you will see four metric cards: Total Orders, Pending, Shipped Today, and Total Revenue. These update automatically as orders are synced.

Platform Status Bar

A row of coloured chips (Shopee, Lazada, TikTok Shop, Shopify) shows which platforms are currently connected. A green dot means live; grey means not connected. Click any chip to jump to Connections.

Order List

The main table lists all orders. Columns include: Order ID, Client, Channel, Date, Status, and Total. Click any row to open the order detail panel.

主页是您的主要工作区，可以一目了然地查看所有订单、关键统计数据和平台连接状态。

统计卡片

顶部显示四个指标卡片：总订单数、待处理、今日发货和总收入。这些数据随订单同步自动更新。

平台状态栏

一排彩色标签（Shopee、Lazada、TikTok Shop、Shopify）显示当前已连接的平台。绿点表示已连接，灰色表示未连接。点击任意标签可跳转至渠道连接页面。

订单列表

主表格列出所有订单，列包括：订单编号、客户、渠道、日期、状态和金额。点击任意行可打开订单详情面板。

💡
Use the search box in the header to quickly find orders by ID, client name, or tracking number. 使用标题栏中的搜索框，可通过订单编号、客户名称或追踪号快速查找订单。
Chapter 4

Platform Connections

渠道连接

IdealOne OMS connects to four e-commerce platforms: Shopee, Lazada, TikTok Shop, and Shopify. Navigate to Connections in the sidebar to manage these.

Connecting a Platform

  1. Click on the platform tile (e.g., Shopee).
  2. Enter the required credentials — each platform needs different keys (see below).
  3. Click Save. The tile turns blue when connected.
  4. Click Sync Now to fetch the first batch of orders.

Platform Credentials

IdealOne OMS 连接四个电商平台：Shopee、Lazada、TikTok Shop 和 Shopify。在侧边栏导航至渠道连接进行管理。

连接平台

  1. 点击平台磁贴（如 Shopee）。
  2. 输入所需凭据——每个平台需要不同的密钥（见下文）。
  3. 点击保存。连接成功后磁贴变为蓝色。
  4. 点击立即同步获取第一批订单。

平台凭据说明

PlatformRequired Fields / 所需字段
ShopeePartner ID, Partner Key, Shop ID, Access Token
LazadaApp Key, App Secret, Access Token

ShopifyStore Name (myshopify.com subdomain), Access Token

Chapter 5

Order Management

订单管理

All orders — whether synced from platforms or manually entered — appear on the Dashboard order list.

Filtering Orders

Use the filter bar above the order list to filter by:

  • Status — Pending, Confirmed, Processing, Packed, Shipped, Cancelled
  • Channel — Shopee, Lazada, TikTok Shop, Shopify, Manual
  • Client — Select a client from the left sidebar
  • Date Range — Filter by order date

Adding Orders Manually

  1. Click + Add Order in the header.
  2. Fill in Client, Channel, Order ID, and items.
  3. Click Save. The order appears immediately in the list.

Order Details Panel

Click any order row to open the side panel. Here you can view order items, shipping details, notes, and update the order status manually.

所有订单——无论是从平台同步的还是手动录入的——都显示在主页订单列表中。

筛选订单

使用订单列表上方的筛选栏按以下条件筛选：

  • 状态 — 待处理、已确认、处理中、已打包、已发货、已取消
  • 渠道 — Shopee、Lazada、TikTok Shop、Shopify、手动录入
  • 客户 — 从左侧边栏选择客户
  • 日期范围 — 按订单日期筛选

手动录入订单

  1. 点击标题栏的+ 新建订单。
  2. 填写客户、渠道、订单编号和商品信息。
  3. 点击保存，订单立即出现在列表中。

订单详情面板

点击任意订单行打开侧边面板，可查看订单商品、配送详情、备注，并手动更新订单状态。

Chapter 6

Sync Log

同步记录

The Sync Log records every synchronisation event — whether triggered automatically or manually. Navigate to Sync Log in the sidebar.

Reading the Log

Each log entry shows:

  • Platform — Which platform was synced
  • Time — When the sync ran
  • Fetched — Total orders retrieved from the platform
  • Added — New orders added to the local database
  • Error — Any error message (empty if successful)

Triggering a Manual Sync

Click the Sync All button in the header or the individual Sync Now button on each platform tile in Connections.

同步记录记录每次同步事件——无论是自动触发还是手动触发。在侧边栏导航至同步记录。

读取同步日志

每条日志记录显示：

  • 平台 — 同步的平台
  • 时间 — 同步执行时间
  • 获取 — 从平台获取的订单总数
  • 新增 — 添加到本地数据库的新订单数
  • 错误 — 错误信息（成功时为空）

触发手动同步

点击标题栏的全部同步按钮，或在渠道连接页面的各平台磁贴上点击立即同步按钮。

Chapter 7

Fulfillment Pipeline

发货流程

Navigate to Fulfillment in the sidebar to manage the order fulfillment workflow. Orders move through five statuses:

在侧边栏导航至发货管理，管理订单履行工作流。订单经过五个状态：

Pending 待处理 → Confirmed 已确认 → Processing 处理中 → Packed 已打包 → Shipped 已发货

Status Descriptions

  • Pending — Order just received; awaiting confirmation.
  • Confirmed — Order verified and accepted.
  • Processing — Order being picked and prepared.
  • Packed — Items packed; inventory deducted automatically at this stage.
  • Shipped — Parcel handed to courier; platform notified automatically.

Fulfilling an Order

  1. Find the order in the Fulfillment list (filter by status if needed).
  2. Click Fulfill to advance it to the next status.
  3. At the Packed stage, inventory is deducted automatically.
  4. At the Shipped stage, the platform API is updated automatically.

Bulk Fulfillment

Select multiple orders using the checkboxes, then click Bulk Fulfill to advance all selected orders to the next status at once.

Printing Waybills

Click the Print Waybill button on any order to fetch and print the shipping label from the respective platform.

状态说明

  • 待处理 — 刚收到订单，等待确认。
  • 已确认 — 订单已核实并接受。
  • 处理中 — 订单正在拣货和备货。
  • 已打包 — 商品已打包；此阶段自动扣减库存。
  • 已发货 — 包裹已交给快递；自动通知平台。

履行订单

  1. 在发货管理列表中找到订单（如需要可按状态筛选）。
  2. 点击发货将其推进到下一状态。
  3. 在"已打包"阶段，库存自动扣减。
  4. 在"已发货"阶段，平台 API 自动更新。

批量发货

使用复选框选择多个订单，然后点击批量发货，一次性将所有选中订单推进到下一状态。

打印运单

点击任意订单上的打印运单按钮，从相应平台获取并打印快递面单。

Chapter 8

IDEALSCAN

IDEALSCAN is the built-in barcode scanning module. Navigate to IDEALSCAN in the sidebar. It lets you scan order barcodes with your device camera and instantly advance the order status.

Camera Setup

  1. Click Start Camera. Your browser will ask for camera permission — click Allow.
  2. If you have multiple cameras, use the Switch Camera button to select the best one.
  3. Point the camera at an order barcode. When recognised, the order pops up automatically.

Scanning Workflow

  1. Scan the barcode on the parcel. The matching order is displayed with its current pipeline stage.
  2. Review the order details (client, channel, items).
  3. Click Update Status to advance to the next stage.
  4. A beep and flash confirms the update. The order is saved to the session history below.

Manual Entry

If the barcode is unreadable, type the order ID or tracking number into the manual entry field and press Enter.

Session History

All scanned orders in the current session are listed below the scanner. You can export the session as a CSV file using Export CSV.

Bulk Scan Fulfillment

After scanning multiple orders, click Bulk Fulfill All to advance all session orders at once.

IDEALSCAN 是内置的条形码扫描模块。在侧边栏导航至扫码发货。它可以用设备摄像头扫描订单条形码，并即时更新订单状态。

摄像头设置

  1. 点击启动摄像头，浏览器会请求摄像头权限——点击允许。
  2. 如果有多个摄像头，使用切换摄像头按钮选择最佳摄像头。
  3. 将摄像头对准订单条形码，识别后订单自动弹出。

扫描工作流

  1. 扫描包裹上的条形码，显示匹配订单及其当前流程阶段。
  2. 查看订单详情（客户、渠道、商品）。
  3. 点击更新状态推进到下一阶段。
  4. 蜂鸣声和闪光确认更新成功，订单保存到下方的会话历史记录。

手动录入

如果条形码无法识别，在手动录入框中输入订单编号或追踪号，然后按 Enter。

会话历史记录

当前会话中扫描的所有订单都列在扫描仪下方。可以使用导出 CSV将会话导出为 CSV 文件。

批量扫描发货

扫描多个订单后，点击批量履行全部一次性推进所有会话订单。

💡
IDEALSCAN works best in a well-lit environment. Use the rear camera of a phone or tablet for fastest barcode recognition. IDEALSCAN 在光线充足的环境中效果最佳。使用手机或平板的后置摄像头可获得最快的条形码识别速度。
Chapter 9

Inventory Management

库存管理

Navigate to Inventory in the sidebar to manage your product stock. Inventory is automatically deducted when orders reach the Packed status.

Stats Overview

At the top you will see: Total SKUs, Low Stock (items at or below reorder point), Out of Stock, and Total Inventory Value.

Adding a Product

  1. Click + Add Product.
  2. Enter the SKU, Name, Category, Unit (pcs/kg/box), initial Stock Qty, Reorder Point, Cost Price, and Sell Price.
  3. Click Save.

Stock Adjustments

Click any product row to open the side panel. Use the adjustment form to:

  • Stock In — Add received stock (purchases, returns)
  • Stock Out — Manual deductions (damage, loss)
  • Adjustment — Correct discrepancies from stock-take

Movement History

The side panel also shows a full movement log for each SKU — every in/out transaction with timestamp and reason.

Low Stock Alert

Items at or below the Reorder Point appear highlighted. The orange badge on the Inventory nav item shows the count of low-stock items.

在侧边栏导航至库存管理管理产品库存。当订单达到已打包状态时，库存自动扣减。

统计概览

顶部显示：SKU 总数、低库存（达到或低于补货点的商品）、缺货和库存总价值。

添加产品

  1. 点击+ 添加产品。
  2. 输入 SKU、名称、类别、单位（件/千克/箱）、初始库存数量、补货点、成本价和售价。
  3. 点击保存。

库存调整

点击任意产品行打开侧边面板，使用调整表单进行：

  • 入库 — 添加收到的库存（采购、退货）
  • 出库 — 手动扣减（损坏、丢失）
  • 盘点调整 — 修正盘点差异

移动历史记录

侧边面板还显示每个 SKU 的完整移动日志——每笔入库/出库记录，含时间戳和原因。

低库存提醒

达到或低于补货点的商品会高亮显示。库存导航项上的橙色徽章显示低库存商品数量。

Chapter 10

Client Portal

客户门户

The Client Portal (/portal) is a dedicated view for your merchant clients. Clients can log in to track their own orders without accessing the full OMS.

What Clients Can See

  • Summary stats (total orders, pending, shipped today, revenue)
  • Order list filtered to their account only
  • Order details (items, shipping info, current status)
  • Filter by date range, status, and channel
  • Export orders as CSV

Client Report in OMS

Administrators can also view the Client Report from within the OMS by clicking Client Report in the sidebar. Use the client selector at the top to choose which client's data to view.

客户门户（/portal）是专为商家客户提供的专属视图。客户可以登录查看自己的订单，而无需访问完整的 OMS 系统。

客户可以查看的内容

  • 摘要统计（总订单数、待处理、今日发货、收入）
  • 仅限其账户的订单列表
  • 订单详情（商品、配送信息、当前状态）
  • 按日期范围、状态和渠道筛选
  • 导出 CSV 格式订单

OMS 中的客户报表

管理员也可以在 OMS 内点击侧边栏的客户报表查看客户报表，使用顶部的客户选择器选择要查看的客户数据。

Chapter 11

Administrator Settings

管理员设置

Navigate to Administrator in the sidebar to manage users, API keys, and system settings.

Client Users

Manage who can log into the Client Portal:

  • Add User — Create a new client login (assign to a client account)
  • Reset Password — Change a user's password
  • Disable/Enable — Temporarily block or restore access
  • Delete — Permanently remove a user

API Keys

API keys allow programmatic access to the IdealOne OMS API. Useful for integrations with third-party tools.

  • Click Generate Key to create a new API key for a client.
  • Copy the key immediately — it will not be shown again in full.
  • Revoke a key when it is no longer needed.
⚠️
Keep API keys secret. Treat them like passwords. Never share them in plain text or commit them to version control.

Sync Settings

Auto-sync interval defaults to 15 minutes. Change the SYNC_INTERVAL_MINUTES environment variable to adjust.

在侧边栏导航至管理员，管理用户、API 密钥和系统设置。

客户用户

管理可以登录客户门户的用户：

  • 添加用户 — 创建新的客户登录（分配到客户账户）
  • 重置密码 — 更改用户密码
  • 禁用/启用 — 临时封锁或恢复访问
  • 删除 — 永久删除用户

API 密钥

API 密钥允许以编程方式访问 IdealOne OMS API，适用于与第三方工具的集成。

  • 点击生成密钥为客户创建新 API 密钥。
  • 立即复制密钥——之后将不再完整显示。
  • 不再需要时撤销密钥。
⚠️
保持 API 密钥安全。将其视为密码，切勿以明文分享或提交到版本控制。

同步设置

自动同步间隔默认为 15 分钟。修改 SYNC_INTERVAL_MINUTES 环境变量可调整。
